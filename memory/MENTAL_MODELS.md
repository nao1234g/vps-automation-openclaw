## 判断・意思決定系

### Geneen Backward Planning
```
❌ 積み上げ型: 「できることを積み上げていつかゴールに着く」
✅ 逆算型: 「ゴールを宣言し、そこから逆算して今日やることを決める」

使う場面: プロジェクト計画、タスク優先度付け、年次目標設定
```

### Bezos Regret Minimization
```
「80歳の自分が振り返ったとき、これをやらなかったことを後悔するか？」

使う場面: Type 1判断（取り消せない大きな決断）
```

### 5 Facts Taxonomy（Geneen）
```
Unshakeable facts  = 検証済みデータ ← これだけで判断する
Surface facts      = 見た目はそう見えるが不完全
Assumed facts      = 誰もが真と思っているが未検証
Reported facts     = 誰かがそう言っているもの
Wishful facts      = そうなってほしいもの（最危険）

使う場面: 重要な意思決定の前、AIへの指示の前
```

## 予測・分析系

### Base Rate First
```
「この種のことが過去に何%の確率で起きたか？」を先に調べる。
直感よりも歴史的基準率の方が予測精度が高い。

使う場面: すべてのNowpattern予測の前
```

### Fermi Estimation
```
大きな問いを小さな問いに分解して掛け算する。
「日本でAIを使って予測に参加する人は何人か？」
= 日本のネット人口 × AIリテラシー率 × 予測への関心 × Nowpatternの認知率

使う場面: 市場規模推計、目標設定
```

### Outside View vs Inside View
```
Inside view: 「このプロジェクトは特別だからうまくいく」（バイアスあり）
Outside view: 「この種のプロジェクトは何%が成功するか？」（客観的）

予測精度が高い人は常にOutside viewから始める。
```

## 事業・競争系

### Porter's 5 Forces（簡略版）
```
競合の脅威 + 新規参入 + 代替品 + 買い手交渉力 + 供給者交渉力
= 業界の魅力度

NowpatternへのApply:
- 競合: Metaculus/Manifold（英語のみ → 脅威低）
- 新規参入: 日英バイリンガル予測サービス（現存せず → 参入障壁高い）
- 代替品: SNSでの予測投稿（精度追跡なし → 差別化明確）
```

### Flywheel（Amazon式フライホイール）
```
良いコンテンツ → 読者増加 → 予測参加者増加 → データ蓄積 →
予測精度向上 → より良いコンテンツ → ...（ループ）

Nowpatternのフライホイール:
予測生成 → 公開 → 自動検証 → Brier Score蓄積 →
読者信頼向上 → 読者参加増加 → 精度向上 → ...
```

## 学習・知識系

### First Principles Thinking
```
常識（前提）を取り除き、本当に正しいことだけから再構築する。
「予測サイトはこういうもの」という先入観を捨てて設計する。

使う場面: 新機能設計、問題の根本原因分析
```

### Cross-Domain Connection（最も価値が高いパターン）
```
2つの無関係に見えるドメインの接続点に洞察が宿る。

例: 歴史(帝国衰退) × 経営(競争優位) = 「トップ企業の衰退は突然ではなく10-15年前から兆候がある」
→ Nowpatternの「企業業績予測」の精度向上に直結

使う場面: 壁打ちセッション、新しい予測を立てる前
```
