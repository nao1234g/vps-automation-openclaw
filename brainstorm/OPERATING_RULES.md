# Brainstorm Sessions — 運用ルール

> 「月1回、Nowpatternから主語を取り戻す時間」
> これは制度です。意志の強い週にだけやるものではありません。

---

## 最重要ルール（3行）

```
1. 月に1回やれば十分。完璧なセッションである必要はない。
2. 「Nowpatternの作業進捗報告」にしない。プロジェクト外の問いを持ち込む。
3. 終わったら SESSION_INDEX.md に1行追加する。それだけが必須。
```

---

## セッションのやり方

### 準備（1分）

1. `sessions/SESSION_TEMPLATE.md` を開く
2. 今月の問いを1つ決める（下の「テーマ選びのヒント」を参照）
3. `sessions/YYYY-MM-DD_テーマ.md` という名前でファイルを作る

### 実施（30〜60分）

- AIに「壁打ちしたい」と伝える
- テンプレートの問いを順番に考える
- 「決めたこと」と「保留したこと」を区別して記録する

### 終了（5分）

必ずやること（省略禁止）:

| アクション | ファイル |
|-----------|---------|
| セッションを1行で記録 | `SESSION_INDEX.md` |
| 新しいクロスドメイン洞察があれば追記 | `../intelligence/DOMAINS.md` |
| 重要な意思決定があれば記録 | `../decisions/DECISION_LOG.md` |

---

## テーマ選びのヒント

**よいテーマの条件**（3つのうち1つ以上当てはまれば使える）:
- Nowpatternの作業中に「これは問いが深い、後で考えたい」と感じたこと
- 「5年後どうなりたいか」に関係すること
- 認知の歪みや思い込みを確かめたいこと

**テーマの例**:
- 「自分が最も外れやすい予測のパターンは何か？」
- 「Nowpatternが5年後に成功しているとき、何が起きているか？」
- 「今週、Nowpatternに引っ張られて考えなかったことは何か？」
- 「今の自分の世界観のどこに、最も大きな盲点があるか？」
- 「予測精度を上げるために、次に学ぶべき分野はどこか？」

**テーマを決められないときは**: 前回のセッションの「次回の宿題」を使う。

---

## よくある誤解と対処

| 誤解 | 現実 |
|------|------|
| 「深い洞察がないとセッションとして価値がない」 | 30分考えて結論が出なくても、問い続けたことに価値がある |
| 「毎月やらないと失敗」 | 2ヶ月空いても、再開すればよい。制度は壊れない |
| 「Nowpatternと関係ないテーマはここでやるべきでない」 | むしろNowpatternと直接関係ないテーマこそここでやる |
| 「完璧なセッションを作ってから公開する」 | 未完成でもファイルを作れば積み上がる。整理は後でいい |

---

## スケジュール（最低限）

```
毎月第1週の月曜: brainstorm session（30〜60分）
  → Nowpatternの週次タスクより先にカレンダーに入れる
  → "月次OS点検" と名付けると忘れにくい
```

**次回予定**: 2026年4月 第1週（月）

## ファイル構成

```
brainstorm/
  OPERATING_RULES.md      ← このファイル（制度ルール）
  SESSION_TEMPLATE.md     ← 毎回コピーして使うテンプレート
  SESSION_INDEX.md        ← 全セッションの一覧（必ず更新）
  sessions/               ← 実際のセッションファイル置き場
    YYYY-MM-DD_テーマ.md
```

## 成功基準

このbrainstorm制度が「機能している」状態:

```
✅ SESSION_INDEX.md に毎月1件追加されている
✅ 毎回「決めたこと」が1つ以上記録されている
✅ DOMAINS.md の洞察が増えている
✅ 「Nowpatternの作業報告」になっていない
```
