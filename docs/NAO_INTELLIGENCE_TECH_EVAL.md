### 軸01: Observer/Reflector パターン（Mastra OM方式）

**何をするか**: バックグラウンドObserverエージェントが実行を継続観察 → 観察ログをappend-only蓄積 → 定期ReflectorがログをWISDOMに変換

**ベンチマーク**: LongMemEval 94.87%（gpt-5-mini）/ 84.23%（gpt-4o）

**評価**:
| 評価軸 | スコア | 根拠 |
|--------|--------|------|
| 精度向上 | ★★★★★ | 54%→94%（+40pt）は圧倒的 |
| 実装コスト | ★★★★☆ | Pythonスクリプトで実装可能、新インフラ不要 |
| Nao Intelligence適合 | ★★★★★ | `AGENT_WISDOM.md` の手動更新を自動化する |
| コスト追加 | ★★★★☆ | プロンプトキャッシュで4〜10倍削減（追加コストなし） |

**判定: A（今すぐ採用）**

**実装案**:
```python
# /opt/shared/scripts/memory_observer.py（新規作成）
# NEO-ONE/TWO の各タスク実行後にフックで呼び出し
# observations.log（append-only）に記録
# 50件蓄積でReflectorがAGENT_WISDOM.mdを更新
```

**ベンチマーク**: LongMemEval 91%（低レイテンシ版）

**評価**:
| 評価軸 | スコア | 根拠 |
|--------|--------|------|
| 精度向上 | ★★★★☆ | 91%は実用的 |
| 実装コスト | ★★★★★ | `pip install mem0`、1行でエージェント統合 |
| Nao Intelligence適合 | ★★★★★ | NEO-ONE/TWO のsdk_integration.pyに追加可能 |
| コスト追加 | ★★★★☆ | OSS版はセルフホスト可（追加コストなし）|

**判定: A（今すぐ採用）**

**実装案**:

# /opt/claude-code-telegram/sdk_integration.py に追加
from mem0 import Memory
m = Memory()

m.add(f"タスク: {task_name}, 結果: {result}, 学習: {learning}", agent_id="neo-one")
# 次タスク開始時
memories = m.get_all(agent_id="neo-one")

### 軸06: Reflexion（タスク後の言語反省）

**何をするか**: タスク実行後、エージェントが「何が悪かったか」を言語で反省 → 次回タスクの精度向上。

**ベンチマーク**: 精度20%以上向上（Stanford研究）

**評価**:
| 評価軸 | スコア | 根拠 |
|--------|--------|------|
| 精度向上 | ★★★★★ | 20%向上、追加インフラゼロ |
| 実装コスト | ★★★★★ | プロンプト追加のみ（コードなし）|
| Nao Intelligence適合 | ★★★★★ | NEO-ONE/TWO の全タスク後に追加可能 |
| コスト追加 | ★★★★★ | ゼロ（Claude Max定額内）|

**判定: A（今すぐ採用）**

**実装案**:
```python
# NEO-ONE/TWO のsystem_promptに追加
REFLECTION_SUFFIX = """
タスク完了後、必ず以下を実行:
1. 「何がうまくいったか」「何がうまくいかなかったか」を1〜2文で記録
2. 次回同じタスクをやるならどう改善するか
3. /opt/shared/task-log/YYYY-MM-DD_reflection.md に追記
"""
```
