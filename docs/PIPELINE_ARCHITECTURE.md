## 1. 目的 — 何のためにやるのか

### ミッション

> **ニュースの裏側に繰り返し現れる「パターン（力学）」を発見し、名前をつけ、世界に届ける測定装置を作る。**

### 核心の価値方程式

```
記事数 × 差分密度 × 到達人数 = ブランド価値 → マネタイズ
```

- **記事数**: 200記事/日（JP 100 + EN 100）で圧倒的な在庫
- **差分密度**: Deep Pattern形式で「なぜ・次はどうなるか」を分析
- **到達人数**: X 100投稿/日 + note + Substackで配信

### 3段階ゴール

| フェーズ | 期間 | 目標 | KPI |
|---------|------|------|-----|
| **Phase 1: 量産** | 月1〜3 | 200記事/日 + X100投稿/日で認知度を作る | Ghost記事数6,000+、Xフォロワー5,000+ |
| **Phase 2: 有料化** | 月4〜6 | 記事の後半を有料化、Xサブスク開始 | Ghost有料会員200+、MRR $1,000+ |
| **Phase 3: 拡大** | 月7〜12 | 4チャネル収益化で月$5,000+を安定的に稼ぐ | MRR $5,000+、Substack有料500+ |

## 2. 記事の構造 — 「どこまで無料、どこから有料」

### 2A. 記事の全体像（12セクション）

```
┌──────────────────────────────────────────────────┐
│                                                    │
│  ★ 無料ゾーン（誰でも読める）                       │
│  ─────────────────────────────────────             │
│  0. FAST READ         1分サマリー                   │
│     - 逆説フック（1文でハッとさせる）                │
│     - 3点箇条書き（事実3つ）                        │
│     - NOW PATTERNタグ + 力学の一文                  │
│     - 確率と対応（3シナリオ × 各1行）               │
│     - CTA「なぜ？ → 続きを読む ↓」                 │
│                                                    │
│  1. BOTTOM LINE       3秒で核心                    │
│  2. DELTA             前回からの変化（確率テーブル） │
│  3. タグバッジ         ジャンル / イベント / 力学    │
│  4. Why it matters    なぜ重要か（2〜3文）          │
│  5. What happened     事実リスト（5〜8項目）        │
│                                                    │
│  ここまでで「何が起きたか」は完全にわかる            │
│                                                    │
├──────────────── ★ 有料の壁 ★ ──────────────────────┤
│                                                    │
│  ★ 有料ゾーン（$9.99/月の会員のみ）                 │
│  ─────────────────────────────────────             │
│  6. The Big Picture   歴史的文脈 + 利害関係者マップ  │
│  7. Between the Lines 報道が言っていないこと         │
│  8. NOW PATTERN       力学分析 × 2 + 交差点         │
│  9. Pattern History   過去の並行事例 × 2            │
│  10. What's Next      3シナリオ + トリガー           │
│  11. OPEN LOOP        次のトリガー + 追跡テーマ      │
│                                                    │
│  ここが「なぜそうなったか」「次はどうなるか」        │
│  = Nowpatternの本当の価値 = お金を払う理由           │
│                                                    │
└──────────────────────────────────────────────────┘
```

### 2B. なぜこの線引きか

**読者の心理を考える:**

1. FAST READ（無料）で「ハッ」とする → 「もっと知りたい」
2. What happened（無料）で事実を知る → 「でも、なぜ？」
3. **ここで有料の壁** → 「なぜ」を知りたいなら$9.99/月
4. Big Picture〜What's Next（有料）で構造と予測を読む

**無料ゾーンだけでも十分な価値がある**:
- 1分で何が起きたかわかる（FAST READ）
- 事実リストで全体像がつかめる（What happened）
- これだけで普通のニュースサイトと同等

**有料ゾーンはNowpatternにしかない価値**:
- 歴史的文脈と利害関係者マップ（The Big Picture）
- 報道されない裏側（Between the Lines）
- 力学分析（NOW PATTERN）← これが核心
- 過去の類似パターン（Pattern History）
- 3つのシナリオ予測（What's Next）

### 2C. Phase 1は全部無料

**最初の3ヶ月は有料の壁を設けない。全文無料。**

理由:
- まず読者を集めないと有料化しても誰も買わない
- 200記事/日の量で「Nowpatternは分析がすごい」という評判を作る
- 6,000記事の在庫ができたPhase 2で有料化する

## 3. 記事ルール — すべての記事に適用される法律

### 3A. 基本ルール

| ルール | 内容 |
|--------|------|
| フォーマット | **Deep Pattern一択**（Speed Log廃止） |
| 言語 | 1記事1言語。日本語と英語の混在禁止 |
| タイトル | `[固有名詞 + 出来事] — [独自の視点]`（60文字以内） |
| タイトル禁止 | 「観測ログ」「Speed Log」「Deep Pattern」「#番号」 |
| フッター | `Tags:` を手書きしない（スクリプトが自動追加） |

### 3B. タクソノミー v3.0（3層構造）

| レイヤー | 個数 | 1記事あたり | 用途 |
|----------|------|-------------|------|
| **ジャンル** | 13 | 1〜2個 | サイトナビゲーション |
| **イベントタグ** | 19 | 1〜2個 | 記事フィルター |
| **力学タグ** | 16（4×4） | 1〜3個 | NOW PATTERN分析セクション |

力学タグの4ファミリー:
- **支配**: プラットフォーム支配 / 規制の捕獲 / 物語の覇権 / 権力の過伸展
- **対立**: 対立の螺旋 / 同盟の亀裂 / 経路依存 / 揺り戻し
- **崩壊**: 制度の劣化 / 協調の失敗 / モラルハザード / 伝染の連鎖
- **転換**: 危機便乗 / 後発逆転 / 勝者総取り / 正統性の空白

### 3C. 執筆フロー（1ネタ → JP記事 + EN記事 = 2本）

```
NEO-ONE/NEO-TWO が執筆キューから1件取得
  ↓
1. 日本語記事を執筆（Deep Pattern形式）
  ↓
2. Ghost APIで日本語版を公開（tag: lang-ja）
  ↓
3. 同じNEOが自分で英語翻訳（外部モデル不使用）
  ↓
4. Ghost APIで英語版を公開（tag: lang-en）
  ↓
5. X投稿キューに追加
  ↓
6. note/Substackキューに追加
```

**1ネタから2記事（JP + EN）が生まれる。100ネタ/日 = 200記事/日。**

## 4. 容量計算 — 100記事/日は本当に可能か

### 4A. 基本リソース

| リソース | 値 |
|----------|-----|
| Claude Max アカウント | 2つ（個人 + 会社） |
| 月額 | $200 × 2 = $400/月（定額） |
| 1アカウントのOpus稼働 | 約40時間/週 |
| 合計 | 80時間/週 = **約11.4時間/日 = 685分/日** |

### 4B. 並列実行

NEO-ONE と NEO-TWO は同時に動く:

```
NEO-ONE（アカウント1: 個人）: 342分/日
NEO-TWO（アカウント2: 会社）: 342分/日
合計: 685分/日
```

### 4C. 1記事あたりの所要時間を短縮する

**従来（NEOが自分でリサーチ → 執筆 → 翻訳）**: 約15分/ペア
→ 685分 ÷ 15分 = 45ペア/日 ← これだと足りない

**パイプライン最適化後（ネタプールから構造化データを受け取って執筆）**:

パイプラインがNEOに渡すデータ:
- ネタのタイトル、ソースURL、関連ツイートID
- Gemini/Grokが抽出した事実リスト
- 関連する過去記事（DELTA用）
- 推奨タグ（ジャンル/イベント/力学）

NEOがやること:
- 構造化データから記事を組み立てる（リサーチ不要）
- JP記事: **約4分**
- EN翻訳: **約2分**
- 合計: **約6分/ペア**

→ 685分 ÷ 6分 = **114ペア/日** ← 100を超える

### 4D. 安全マージン込みの計算

| 項目 | 値 |
|------|-----|
| 利用可能時間 | 685分/日 |
| ローカルClaude Code使用分 | -60分/日 |
| バッファ（エラー、待機） | -85分/日 |
| 実質稼働 | **540分/日** |
| 1ペア所要時間 | 5分（JP 3分 + EN 2分、最適化後） |
| **100ペア必要時間** | **500分** |
| **余裕** | **40分** |

**結論: パイプラインからの構造化データ供給があれば、100ペア/日（= 200記事）は達成可能。**

```
NEO-ONE: アカウント1 → 上限到達 → アカウント2に自動切替
NEO-TWO: アカウント2 → 上限到達 → アカウント1に自動切替
```

## 5. Xルール — 1日100投稿の設計

### 5A. 投稿フォーマット（5種類）

| フォーマット | 件数/日 | 内容 | ボットリスク |
|-------------|---------|------|-------------|
| **引用リポスト** | 80件 | ネタプールのmatched_tweetを引用 | 低 |
| **画像+オリジナル** | 10件 | ツイート見つからない場合 | 中 |
| **X Article** | 1件 | 最重要記事の全文（25,000文字） | 低 |
| **スレッド** | 1件 | 記事要約を5-8ツイートに分割 | 低 |
| **リプライ** | 8件 | 賛同のみ、0件の日もOK | 最低 |
| **合計** | **100件** | | |

### 5B. 投稿テキストのパターン（3種類）

**どのフォーマットで投稿するかとは別に、テキストの書き方が3パターンある。**

#### P1: 好奇心ギャップ型（デフォルト — 新規トピック向け）

```
DeepSeekが3日でGPT-4oを超えたのに、なぜアメリカは笑っているのか。

Full analysis ↓
https://nowpattern.com/deepseek-leapfrog/

#Nowpattern #ニュース分析 #DeepSeek
```

**使うとき**: 新しいトピック。世間の解釈が間違っている。「え、なんで？」と思わせたい。

#### P2: 差分提示型（Delta — 前回記事があるトピック向け）

```
トランプ関税の悲観シナリオ、20% → 35%に急上昇。

前回（2/21）から何が変わったのか。
Full analysis ↓
https://nowpattern.com/trump-tariff-delta/

#Nowpattern #ニュース分析 #Tariffs
```

**使うとき**: 前回記事がある。シナリオ確率が変わった。数字の変化で「お、動いた」と思わせたい。

#### P3: 損失回避型（投資・行動判断直結のトピック向け）

```
EU新規制、7月施行。対応しないテック企業は売上の6%を罰金で失う。

あなたのポートフォリオに影響する3シナリオ ↓
https://nowpattern.com/eu-dma-penalty/

#Nowpattern #ニュース分析 #EU
```

**使うとき**: 投資判断や行動に直結する。長期トレンドの分岐点。「これ見逃すとヤバい」と思わせたい。

### 5C. ハッシュタグルール（コードで強制済み）

```
必須①: #Nowpattern（ブランド）    ← x-auto-post.py が自動追加
必須②: #ニュース分析（カテゴリ）  ← x-auto-post.py が自動追加
追加:   固有名詞タグ 1〜2個（#DeepSeek #AI #EU 等）
合計: 3〜4個

⚡ 強制: x-auto-post.py の MANDATORY_HASHTAGS で物理的に追加される。
         NEOが忘れても、コードが必ず入れる。

❌ 禁止:
- 内部タクソノミータグ（#後発逆転 #プラットフォーム支配 等）→ Ghost専用
- 数字タグ（#17%下落 等）→ 誰も検索しない。数字は本文に書く
```

### 5D. リプライのルール

```
対象: AI/テクノロジー/地政学のインフルエンサーの投稿
条件: 本当に賛成できるもののみ
内容: 建設的な賛同 + 自分の分析を1-2文添える

❌ 絶対禁止:
- 議論・反論・批判
- エンゲージメントがマイナスになるリプライ
- 無理にリプライしない（0件の日もOK）
```

### 5E. 投稿スケジュール

```
稼働: JST 08:00〜22:00（14時間）
間隔: ランダム 8〜20分（均一間隔はボット判定リスク）
深夜: 22:00-08:00 は投稿しない

時間帯別:
  08:00-10:00  朝の速報           約20件
  10:00-14:00  午前の安定投稿      約30件
  14:00-18:00  午後のピーク        約30件
  18:00-22:00  夜のまとめ          約20件
```

### 5F. ボット対策

| ルール | 内容 |
|--------|------|
| ランダム間隔 | 8〜20分（均一間隔禁止） |
| コンテンツ多様性 | 5フォーマット + 3パターンを混在 |
| 深夜投稿禁止 | 22:00-08:00は一切しない |
| 同一内容禁止 | 同じテキストの重複はAPIが403を返す |
| 引用元チェック | 無関係ツイートの引用禁止 |
| リプライは賛同のみ | 議論は一切しない |

## 6. 記事 ↔ X 連動ルール

### 6A. 1本の記事からX投稿がどう生まれるか

```
[100ネタ/日]
  ↓
[NEOが記事執筆] → Ghost公開（JP + EN = 200記事/日）
  ↓
[X投稿キュー] → 以下のルールで振り分け

通常のネタ（90件）:
  ├─ ツイート見つかった → 引用リポスト × 1
  └─ ツイート見つからない → 画像+オリジナル × 1
  = 90投稿

最重要ネタ（1件/日）:
  ├─ 引用リポスト × 1
  ├─ X Article × 1
  └─ スレッド × 1
  = 3投稿

リプライ（記事テーマに関連）:
  = 7〜8投稿

合計: 90 + 3 + 7 = 100投稿/日
```

### 6B. 記事の要素 → X投稿への変換

| 記事のどの部分 | Xでどう使うか |
|---------------|-------------|
| FAST READ逆説フック | → 引用リポストの1行目（P1好奇心ギャップ型） |
| DELTA確率変化 | → 引用リポストの1行目（P2差分提示型） |
| What's Nextシナリオ | → 引用リポストの1行目（P3損失回避型） |
| FAST READ 3点箇条書き | → スレッドのツイート2-4 |
| NOW PATTERNタグ | → ツイート本文に英語名で記載 |
| チャート/データ | → 画像投稿として使用 |
| 記事全文 | → X Article（1日1本） |
| 記事URL | → 全投稿の末尾に掲載 |

### 6C. 記事が先、Xは配信手段

```
情報収集（Grok + Gemini）→ ネタプール → 執筆キュー
  ↓
NEOが記事執筆 → Ghost公開（JP + EN）  ← 本体
  ↓
X投稿キュー → 自動投稿スクリプト      ← 配信
  ↓
note/Substackキュー → 自動配信         ← 配信
```

## 7. パイプライン全体構造（7層）

```
Layer 1: 情報収集（Grok + Gemini、自動、15-30分間隔）
  ↓ ネタプール（100件/日以上のネタを収集）
Layer 2: ネタ選定 + ツイート照合（Grok X検索）
  ↓ 執筆キュー（100件/日、構造化データ付き）
Layer 3: 記事執筆 + 翻訳（NEO-ONE + NEO-TWO 並列）
  ↓ Ghost CMS（JP 100 + EN 100 = 200記事/日）
Layer 4: X投稿（100件/日、5フォーマット × 3パターン）
  ↓ @nowpattern
Layer 5: 配信（note 3〜5本 + Substack 1〜2本 = 厳選のみ）
  ↓ 自動（noteシャドバン対策: 少量+間隔空け）
Layer 6: マネタイズ（4チャネル）
  ↓ 収益
Layer 7: 監視 + 学習ループ
```

### Layer 1: 情報収集

#### 1A. Grok X検索（有料 — $5クレジット）

| 項目 | 値 |
|------|-----|
| モデル | grok-4-1-fast |
| 間隔 | 15分（96回/日） |
| コスト | 約 $0.02/日（$0.60/月） |
| 取得対象 | AI収益報告、技術トレンド、地政学速報 |
| **1日の収集目標** | **50ネタ以上** |

#### 1B. Gemini RSS/GDELT/Reddit/HN（無料）

| 項目 | 値 |
|------|-----|
| モデル | gemini-2.5-pro（無料枠） |
| 間隔 | 30分（48回/日） |
| コスト | $0 |
| 取得対象 | RSS 50フィード、GDELT、Reddit 10サブ、HN Top 50 |
| **1日の収集目標** | **100ネタ以上** |

**合計: 150ネタ/日以上を収集 → うち100ネタを記事化**

#### ネタプールのデータ構造

```json
{
  "id": "uuid",
  "title": "ネタのタイトル",
  "source": "grok_x | gemini_rss | gdelt | reddit | hn",
  "source_url": "https://...",
  "facts": ["事実1", "事実2", "事実3"],
  "suggested_tags": {
    "genre": ["テクノロジー"],
    "event": ["技術進展"],
    "dynamics": ["後発逆転"]
  },
  "matched_tweet_id": "1234567890",
  "matched_tweet_url": "https://x.com/.../status/...",
  "x_content_pattern": "P1 | P2 | P3",
  "related_past_articles": ["slug1", "slug2"],
  "relevance_score": 0.92,
  "status": "new | matched | assigned | published | skipped",
  "created_at": "2026-02-23T10:00:00+09:00"
}
```

**パイプラインがここまでデータを構造化してくれるから、NEOは「組み立て」に集中できる → 1記事5分が可能になる。**

```
NEO-TWO: 50ネタ → JP 50記事 + EN 50記事 = 100記事

```

### Layer 5: 配信（note + Substack = 厳選のみ）

**noteとSubstackは「ショーウインドウ」。全記事を送るのではなく、その日の最高の記事だけ選んで送る。**

| 配信先 | 件数/日 | 選定基準 | リスク対策 |
|--------|---------|----------|-----------|
| **note** | 3〜5本 | relevance_score上位のJP記事 | シャドバン歴あり → 投稿間隔4時間以上、急増禁止 |
| **Substack** | 1〜2本 | relevance_score上位のEN記事 | メール通知 → 多すぎると解除される |

```
200記事/日（Ghost） → スコア上位だけ抽出
  ├─ JP上位 3〜5本 → note自動投稿（4時間間隔）
  └─ EN上位 1〜2本 → Substack自動投稿（朝/夕の2回）
```

**note注意事項（シャドバン対策）:**
- 1日5本を超えない
- 投稿間隔は最低4時間（均一間隔もNG、ランダムに3〜5時間）
- 新規アカウントの場合は最初の2週間は1日1本で「慣らし運転」
- 外部リンク多用を避ける（note内に本文を書き、最後にGhostリンク1つだけ）

### Layer 7: 監視 + 学習

| 項目 | 間隔 | 内容 |
|------|------|------|
| パイプライン死活監視 | 毎時 | closed-loop-check.sh |
| 記事生産数 | 毎日 | JP/EN各何本公開できたか |
| X投稿成功率 | 毎日 | 成功/失敗/スキップ集計 |
