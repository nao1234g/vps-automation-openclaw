## 1. 調査の前提

### なぜ「世界最高」を調べるか

> 「現在32/100」という自己評価は、比較基準がなければ意味がない。
> 世界最高が何をやっているかを知ることで「差分」が見える。差分が見えて初めて「何に投資すべきか」が決まる。

### 調査対象の選定基準

- ✅ 公開ベンチマーク（LongMemEval, GAIA, AgentBench, MemoryBench）で検証済み
- ✅ 生産環境で稼働または信頼できる実験結果がある
- ✅ 2025-2026年の最新状態
- ❌ ホワイトペーパーのみ、未検証の主張は除外

## 2. メモリシステム：世界最高水準マップ

### 2.1 ベンチマーク一覧（LongMemEval — 長期記憶精度）

> LongMemEval = ICLR 2025発表。115,000トークン超の会話履歴で評価。5能力: 情報抽出・複数セッション推論・時系列推論・知識更新・適切な棄却。

| システム | スコア | アーキテクチャ | 生産適合 |
|---------|--------|--------------|---------|
| **Supermemory ASMR** | ~99% (98.60%) | 並列エージェントアンサンブル（8-variant） | ⚠️ 実験的（Supermemory自身が「パロディ」と明示） |
| **Mastra OM（gpt-5-mini）** | 94.87% | Observer/Reflectorエージェント + プロンプトキャッシュ | ✅ 生産対応 |
| **Mastra OM（gpt-4o）** | 84.23% | 同上 | ✅ 生産対応 |
| **Mem0** | 91%（低レイテンシ版） / 49%（フルベンチマーク） | 抽出ファクト + ベクトルストア + 知識グラフ | ✅ 生産対応（41K stars, AWS採用） |
| **Zep/Graphiti** | 71.2% | 時系列知識グラフ | ✅ 生産対応 |
| **Supermemory（本番）** | 81.6% | チャンク + ハイブリッド検索 | ✅ 生産対応 |
| **ファイルシステム（MEMORY.md方式）** | **54.2%** | 静的Markdownファイル | ✅ 現在のNao Intelligence方式 |
| **GPT-4o（ベースライン）** | 30〜70% | フルコンテキスト | 参照値 |

**Nao Intelligence の現状: 54.2%（世界最高の54%）**

### 2.2 アーキテクチャ別特性

**① ファイルシステム方式（現在のNao Intelligence）**
- `MEMORY.md` / `AGENT_WISDOM.md` / `SHARED_STATE.md`
- 強み: ゼロインフラ、人間が読める、バージョン管理可能
- 弱み: 検索不能、更新が手動、セッション間で消える、54.2%が上限

**② ベクトルDB方式（RAG）**
- ベンチマーク: 58.3%
- 強み: 大規模ドキュメントへの意味的検索
- 弱み: 時系列変化・矛盾・知識更新に対応できない

**③ アジェンティックメモリ方式（世界最高水準）**
- Observer/Reflectorパターン（Mastra OM）: 94.87%
- 並列エージェントアンサンブル（ASMR）: 99%
- 強み: 時系列推論・矛盾検知・知識更新を自動処理
- 弱み: レイテンシとコストが高い

### 2.3 Mastra Observational Memory（最も現実的な世界最高）

**なぜ注目するか**: 94.87%の精度を達成しながら、コスト4〜10倍削減という「精度とコストの両立」。

**アーキテクチャ**:
```
Observer Agent（バックグラウンド）:
  → 会話・タスク実行を継続的に観察
  → "観察ログ"をappend-onlyで蓄積

Reflector Agent（定期実行）:
  → 観察ログを整理・統合
  → 矛盾を検知・解決
  → 構造化記憶に変換

プロンプトキャッシュ効果:
  → Stable prefix（システムプロンプト + 観察ログ）= キャッシュヒット
  → トークンコスト4〜10倍削減
```

**Nao Intelligence への接続**: 現在の `AGENT_WISDOM.md` + `SHARED_STATE.md` の更新モデルは、Observer/Reflectorパターンの手動版に相当する。これを自動化するのがMastraのアプローチ。

## 3. 自己修正・反省ループ：世界最高水準マップ

### 3.1 Reflexion（NeurIPS 2023）

**何をするか**: タスク後に言語エージェントが自分の行動を言語で反省 → 次のタスクで精度向上。

| 指標 | 値 |
|------|-----|
| 精度向上幅 | 20%以上（Stanfordの研究）|
| インフラコスト | ゼロ（プロンプトのみ） |
| 限界 | 同一モデルが実行・評価・反省するため確証バイアス |

**現在のNao Intelligence**: `evolution_loop.py`（週1回）が部分的に相当するが、深さが浅い。

### 3.2 Multi-Agent Reflexion（2025年12月）

**改善点**: 複数エージェントが互いの推論を評価 → 確証バイアスを排除。

```
Predictor Agent → Critic Agent 1（別モデル）→ Critic Agent 2（別モデル）→ Synthesizer
```

**現在のNao Intelligence**: 未実装。

### 3.3 2026年のトレンド: Test-Time Reasoningとの統合

- o3モデルが「テスト時の推論計算」を追加 → 困難なタスクで精度が劇的向上
- 予測タスク（地政学・経済）は Test-Time Reasoning の恩恵が最も大きい分野
- **Brier Score改善の最大ドライバーになりうる**

## 4. マルチエージェント協調：世界最高水準マップ

### 4.1 スーパーバイザー/マネージャーパターン（2026年標準）

**パフォーマンス**:
- Supervisor + Worker構成: **アクショナブル推奨率 100%**
- 単体エージェント: **1.7%**（同じ研究での比較）

**Gartner調査**: 2024→2025年でマルチエージェント導入問い合わせが**1,445%増加**。

**構造**:
```
Supervisor（意図分解 + ルーティング）
  → Worker A（予測生成）
  → Worker B（記事執筆）
  → Worker C（品質チェック）
```

**現在のNao Intelligence**: NEO-ONE（CTO）/ NEO-TWO（実行）/ NEO-GPT（バックアップ）は役割分担があるが、**直接通信チャネルなし、共有メモリなし**。真のマルチエージェント協調には至っていない。

## 6. 個人知性OSの先行事例

### 6.1 Letta（MemGPT後継）— 最も近い世界事例

**コンセプト**: LLMをOSとして運用。コアメモリ（RAM）+ アーカイブメモリ（ディスク）の階層。

```
Core Memory（RAM相当）:
  → システムプロンプト内の常時アクティブ情報
  → ユーザーの重要属性、現在の目標

Archival Memory（ディスク相当）:
  → 大量の歴史情報（検索可能）
  → 全予測履歴、全タスクログ
```

**2026年1月のConversations API**: 複数の体験を並列に持ち、共有メモリにアクセス。

**Nao Intelligence への示唆**: `SHARED_STATE.md` と `prediction_db.json` の統合をLetta的な「Core/Archive」構造で再設計する可能性がある。

### 6.2 ChatGPT Memory（2026年1月アップグレード）

- 1年分の会話を長期記憶
- Sources機能: 記憶の元となった会話に遡れる
- **Nao Intelligenceとの差**: ChatGPTの記憶は中央集権（OpenAI管理）。Nao IntelligenceはNaoto所有・自律管理が強み。

### 6.3 Alibaba CoPaw（2026年3月）

- オープンソース個人AIアシスタント
- セッション間の永続記憶
- ローカル or クラウドデプロイ
- **意義**: 個人知性OS市場が商品化段階に入った証拠。差別化が急務。

## 7. 世界事例から得る教訓（Nao Intelligence 専用）

### 教訓1: メモリは「データベース問題」ではなく「推論問題」

> 世界最高のシステムはベクトル検索を捨て、エージェントによる能動的推論に移行した。
> Nao IntelligenceのMEMORY.mdは54.2%。Observer/Reflectorパターンを導入すれば94%へ。

### 教訓2: 反省ループは週1回では遅すぎる

> Reflexion論文: タスク後の即座の言語反省で20%以上改善。
> 現在のevolution_loop.pyは週1回。毎タスク後の軽量反省 + 週次の深い反省が世界標準。

### 教訓3: マルチエージェントは「役割分担」だけでは不十分

> Supervisor/Worker + 共有メモリ + リアルタイム通信が必須。
> NEO-ONE/TWOが今の「役割分担 + Telegram通信」から「共有ワーキングメモリ + 直接通信」に進化する必要がある。

### 教訓4: 予測精度の改善は「より多くの予測」ではなく「Test-Time Reasoning」

> o3レベルの推論時計算を地政学・経済予測に適用すると、Brier Score 0.15以下は現実的。
> Nao IntelligenceはClaudeのOpus 4.6を使っているが、特定の「難しい予測」にのみTest-Time Reasoningを追加適用するアーキテクチャが有効。

### 教訓5: 差別化要素は「記録したデータ」にあり

> ChatGPT/CoPaw等の個人AIアシスタントは一般化方向。
> Nao Intelligenceの差別化は「3年分の予測トラックレコード + Brier Score + タイムスタンプ証明」というコピー不可能なデータ資産。
> このデータをメモリシステムに深く統合することが最重要の戦略。
