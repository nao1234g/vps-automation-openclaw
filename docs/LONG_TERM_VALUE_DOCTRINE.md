## 核心原則（Bezos Day 1 Philosophy）

> "I very frequently get the question: 'What's going to change in the next 10 years?'
>  And that is a very interesting question; it's a very common one.
>  I almost never get the question: 'What's not going to change in the next 10 years?'
>  And I submit to you that that second question is actually the more important of the two."
> — Jeff Bezos

**不変のもの（Nowpatternの不変軸）:**
- 予測は検証されるまで記録される（改ざん不可）
- Brier Scoreは数値として積み上がる（消えない）
- 早く始めたプレイヤーがトラックレコードで優位に立つ（時間は買えない）

**この3つを強化するアクションは長期価値 = 高。それ以外は要検討。**

## LTV（Long-Term Value）7次元スコアリング

### 判定方法

実装・記事・投稿・機能追加の前に、以下7次元で採点する。
各次元: 0〜3点。合計21点満点。**15点未満は実施しない。**

| 次元 | 定義 | 0点 | 1点 | 2点 | 3点 |
|------|------|-----|-----|-----|-----|
| **T（トラックレコード蓄積）** | 予測の記録・検証・Brier Score更新への貢献 | 無関係 | 間接的 | 直接貢献 | データ量を10%+増加 |
| **M（モート強化）** | コピーできない競争優位を広げるか | 消耗品 | 汎用 | Nowpattern固有 | 後発が追えない資産 |
| **Q（品質 / 正確性）** | 情報の正確さ・Brier校正への効果 | 精度低下 | 中立 | 精度向上 | 測定可能な改善 |
| **R（読者参加）** | 読者投票・コミュニティ形成への貢献 | 無関係 | 間接的 | 直接的 | 月1,000票以上に貢献 |
| **S（スケーラビリティ）** | 人手ゼロで拡大できるか | 人手必要 | 一部自動 | ほぼ自動 | 完全自動 |
| **E（英語圏到達）** | EN記事・/en/ページ・英語読者への波及 | JA専用 | JA中心 | JA+EN均等 | EN専用（新市場） |
| **C（コスト妥当性）** | ROI = 効果 ÷ コスト（時間・金銭・API） | コスト > 効果 | 均等 | 効果 > コスト | 効果 >> コスト（5倍以上） |

### スコア判定基準

```
18〜21点 → 即実施（最優先）
15〜17点 → 実施（標準優先度）
12〜14点 → Naotoに確認してから実施
9〜11点  → 代替案を提示
0〜8点   → 却下。理由を記録
```

## 7 Powers フレームワーク（Hamilton Helmer）の適用

Nowpatternのモートを7種の競争優位で分析する。

| Power | Nowpatternへの適用 | 現在の強度 |
|-------|-------------------|-----------|
| **Scale Economics** | 記事数・予測数の増加でコスト/記事が下がる | 弱（まだ小規模） |
| **Network Effects** | 読者が増えるほど投票データの信頼性が上がる | 萌芽（ようやく開始） |
| **Counter-Positioning** | 大手メディアは「予測」を公式見解リスクで避ける→Nowpatternが独占 | 強（ニッチ独占） |
| **Switching Costs** | 読者が3年分のトラックレコードを信頼したら他所に行きにくい | 弱（まだ短い） |
| **Branding** | 「予測の精度で選ばれる」ブランドは一度確立すると揺るがない | 準備中 |
| **Cornered Resource** | Superforecaster個人の協力・独自OTSタイムスタンプ | 中（OTSは独自） |
| **Process Power** | 自動検証・Brier更新・AI進化ループは複製困難 | 強（ECC Pipeline） |

**実装判断の問い:**「これはどのPowerを強化するか？」
Powerを強化しない実装は、競合も同じ速度で追いつける = モートに貢献しない。

## 時間軸別の価値評価

### 「今日が3年後の競争優位を決める」原則

```
短期（0〜6ヶ月）:
  - 今日の記事1本 = 3年後のトラックレコード1件
  - 今日の予測1件 = 3年後のBrier Score 1データポイント
  - 今日の読者投票1票 = 3年後のコミュニティ証拠
  → どれも「消耗品」ではなく「永続資産」として扱う

中期（6ヶ月〜2年）:
  - 記事フォーマットの統一 = SEOとAIクローラー評価の蓄積
  - EN/JA バイリンガル構造 = 英語圏からのリンク集積
  - hreflang + /en/ URL構造 = Google言語シグナルの定着

長期（2〜5年）:
  - 3年分のBrier Score = 他のどのメディアも持っていない資産
  - OTSタイムスタンプ = 「いつ言ったか」の不変証明
  - Superforecasterコミュニティ = 人的ネットワークのモート
```

### やらないことの基準（Opportunity Cost）

以下に当てはまる作業は実施しない（= 機会費用が高い）:

```
❌ トラックレコードを増やさない（見た目の改善、UI磨き、説明文修正）
❌ 6ヶ月後には無意味なバズ対応（瞬間的な話題への乗り換え）
❌ 自動化できない手作業の増加（人手依存フローの拡大）
❌ EN記事を減らすJA専用化（英語圏到達の放棄）
❌ 読者参加の障壁を上げる変更（ログイン必須化、摩擦増加）
```

## Nowpatternが絶対に負けない領域（選択と集中）

| 集中する領域 | 理由 |
|-------------|------|
| 地政学・経済・テクノロジー予測 | 日本語圏で予測トラックレコードを積んでいる競合がいない |
| バイリンガル（JA+EN）同時展開 | Metaculus/Manifoldは英語のみ → アジア圏は空白市場 |
| Brier Scoreによる精度の透明性 | 「当たったこと」を主張するメディアはあるが、数値で証明するメディアは存在しない |
| OTSタイムスタンプ | Bitcoinブロックチェーン上の不変証明 = 後付け修正の排除 |

**この4領域を外れたコンテンツ・機能は「集中を失う」行動。**
