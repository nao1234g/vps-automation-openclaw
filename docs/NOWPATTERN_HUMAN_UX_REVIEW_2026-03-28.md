## 1. ホームページ（JA）— https://nowpattern.com/

### 3秒テスト

**結果: △ 微妙**

- タイトル "Nowpattern" のみでは何のサービスか不明
- meta description「ニュースの裏にある力学を構造化し、3シナリオで未来を予測する」→ 良い説明だが、ページ本文で目立つ位置に表示されていない
- Heroセクションの最初の視認情報: Subscribeボタン → **価値より購読を先に押しつけている**

**改善提案**: Hero上部に「Nowpatternとは何か」を1行で見せる（"予測精度を公開する、世界初のバイリンガル予測プラットフォーム"）

### ナビゲーションの可読性

| ラベル | 分かりやすさ | 問題 |
|--------|------------|------|
| Nowpattern | 🟡 普通 | ブランド名。機能説明なし |
| 予測トラッカー | ✅ 良い | 何をするか明確 |
| 力学で探す | ⚠️ 弱い | 「力学」は一般ユーザーに難解。「タグで記事を探す」の方が明確 |
| About | ✅ 良い | シンプルで明確 |
| EN | ✅ 良い | 言語切替と即座に理解 |

### CTAの強度

| CTA | 配置 | 強度 | 問題 |
|-----|------|------|------|
| Subscribe（Hero） | 最上部 | 🟡 中 | 内容を読む前に購読を求める |
| 予測トラッカーを見る | Disclaimerセクション | ⚠️ 弱い | ページ最下部近く、見逃しやすい |
| View Prediction Track Record（EN） | Disclaimerセクション | ⚠️ 弱い | 同上 |

**改善提案**: 記事一覧の直後または記事カードの中に「この予測を追跡する→」CTAを置く

### 記事リスト表示

**✅ 良い点**:
- 記事タイトルが長くても全文表示（日本語ユーザーに配慮）
- 日付表示で鮮度確認可能
- 記事数が豊富（10件表示 + ページネーション）

**⚠️ 問題点**:
- サムネイル画像なし → テキストのみで視覚的変化がない
- タグバッジが見えているか不明（CSS依存）
- 1件404リンクが混在（信頼性を損なう）

## 2. ホームページ（EN）— https://nowpattern.com/en/

### 3秒テスト

**結果: ❌ 不合格（JS依存）**

- JavaScriptが読み込まれる前: ナビに「予測トラッカー」「力学で探す」という日本語が表示
- JSが動作した後: 英語テキストに変換される（ただし遅延あり）
- 海外ユーザー（特に低速回線）は日本語ナビを見て混乱する可能性

**根本的問題**: サーバーサイドでEN/JAのHTMLを分けるべきだが、現在はクライアントサイドCSS切替。

### EN記事タイトル品質

**✅ 良い点**:
- EN記事タイトルは自然な英語（翻訳が良質）
- 10件すべて200 OK（リンク切れなし）

**⚠️ 問題点**:
- 重複記事が見える（South China Sea記事が同じ内容で2つのスラッグから）
- 「South China Sea Standoff」が2件表示される → 読者が「重複？バグ？」と感じる

## 3. /predictions/ — 予測トラッカー

### 3秒テスト

**結果: ✅ 良い**

- スコアボード（213件/11的中/65%的中率）が最初に目に入る
- 数字が大きく表示されており、サービスの実績が即座に伝わる

### 予測カードのUX

**✅ 良い点**:
- 3シナリオ（楽観/基本/悲観）の確率表示が視覚的に明確
- Nowpatternの予測（青）がPolymarket（紫）と区別されている
- アコーディオン形式でコンパクト

**⚠️ 問題点**:
- 「解決済み」セクション（id="np-resolved"）が存在しない → **的中/外れ実績をどこで見るのか不明**
- OracleStatement内の `#np-resolved` リンクが機能しない → ユーザーが迷子になる
- 191件の予測が全部「追跡中」に見える → 実績（解決済み17件）の可視性がゼロ

**改善提案**: np-resolvedセクションを復元または追加し、「11件的中・6件外れ」の実績を可視化

### 検索・フィルターUX

**✅ 良い点**:
- キーワード検索ボックスが見やすい位置
- カテゴリーボタン（経済・地政学・テクノロジー・金融）でフィルター可能
- "全て"ボタンが初期選択で分かりやすい

**⚠️ 問題点**:
- 191件のリストが長すぎる（スクロールが必要）
- ページネーションがないため、下の予測が発見されにくい

### 3秒テスト

**結果: ✅ 良い**

- ページタイトル「Nowpatternとは | 予測オラクル・メディア」が明確
- 価値提案（競合との比較表）が分かりやすい

### コンテンツ品質

**✅ 良い点**:
- 「競合との差別化」比較表が効果的
- X(@nowpattern)へのリンクがある
- Subscribeボタンのテキスト「予測に参加（無料）」が具体的で良い

**⚠️ 問題点**:
- 「0%の的中率」が表示されていると報告あり → 予測数213件あるのに0%？確認要
- CTAが「予測に参加」となっているが、実際は読者投票もメール購読も同じボタン → 混乱あり

## 5. /taxonomy/ — タクソノミーページ

### 3秒テスト

**結果: 🟡 普通**

- ページタイトル「3層フィルターで記事検索」は分かりやすい
- ただし「タクソノミー」という言葉自体が専門的

**✅ 良い点**:
- 13ジャンル × 19イベント × 16力学の3層構造が図示されている
- タグをクリックすると記事一覧へ
- インタラクティブフィルターUIが存在

**⚠️ 問題点**:
- ページタイトルの「タクソノミー」という言葉が一般ユーザーに難解
- 「3層フィルターで記事検索」という副題があるので、メインラベルを「ジャンル・テーマで探す」に変えると分かりやすい

## 6. フッター全体評価

### 現在のフッター構成

```
[フッターナビ]
タクソノミーガイド → /taxonomy-guide-ja/ → ❌誤リダイレクト
Ghost → ghost.org（プラットフォーム帰属表示）

[フッター外に分散]
X(@nowpattern) → /about/ページ内にのみあり
RSS → /rss/（リンクなし？要確認）
```

### フッターのUX問題

| 問題 | 優先度 |
|------|-------|
| リンクが2件しかない（情報が少なすぎる） | 🟡 P3 |
| 「タクソノミーガイド」が誤ったページに飛ぶ | 🔴 P1 |
| EN版でも日本語「タクソノミーガイド」のまま | 🟡 P3 |
| X(@nowpattern)リンクがフッターにない | 🟡 P3 |
| Subscribeリンクがフッターにない | 🟡 P3 |
| プライバシーポリシー/利用規約リンクなし | 🟢 P4 |

**改善提案**: フッターに最低限「X @nowpattern」「Subscribe」「Privacy Policy」を追加
