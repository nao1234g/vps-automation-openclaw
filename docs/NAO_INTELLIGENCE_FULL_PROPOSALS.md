# Nao Intelligence — 全提案リスト（Track 6）
> 作成: 2026-03-26 | セッション継続（完全書き直し）
> 目的: 全提案をドメイン×時間軸で分類し、実装可能性で切る。
> 原則: 事実・仮定・提案を分離。Wishful factsで提案を書かない。

---

## 分類の構造

**2軸分類**:
- **時間軸**: 今すぐ / 次に検証 / 中長期（保留）
- **ドメイン**: 記憶 / 予測 / 実行 / 反省 / オペレーター整合 / Nowpattern公開面 / クロスエージェント

各提案のフォーマット:
```
問題: 何が起きているか（事実）
重要度: なぜ今対処が必要か
実装難度: Low / Medium / High（VPSへのSSHで完結するか）
学習価値: どのタイプの学習（Track 5）に貢献するか
リスク: 副作用・失敗時のダメージ
前提条件: これを実施するために必要なもの
```

- **重要度**: L2記憶スコアが20→85に改善する最大レバー（Track 4参照）

- **保留理由**: データ主権の懸念。Self-hosted版の評価が先
- **評価基準**: VPS上でのself-hosted動作確認後に再評価

### H2-MEM: Zep/Graphiti（時系列知識グラフ）

- **問題**: 同じトピックで矛盾する予測を自動検知できない
- **重要度**: 予測の一貫性確保・矛盾検知はプラットフォーム信頼性に直結
- **実装難度**: High（Docker Community Edition + VPSリソース確認）
- **学習価値**: タイプ1（新事実の構造化）、タイプ8（長期価値判断の知識グラフ化）
- **リスク**: VPS リソース消費（メモリ/CPU）。既存コンテナとの競合確認要
- **保留理由**: VPS リソース制約。不要コンテナ削除後に再評価
- **前提条件**: `docker stats` でリソース余裕の確認 + Mem0 導入との優先順位決定

### H3-XAGENT: Multi-Agent Reflexion（NEO-ONE が予測 → NEO-TWO が批評）

- **問題**: 現状 NEO は自己批評のみ。別エージェントからの批評がない
- **重要度**: Stanford 研究では Multi-Agent Reflexion が単独 Reflexion より精度向上
- **実装難度**: High（エージェント間通信プロトコル設計 + タスクオーケストレーション）
- **学習価値**: タイプ3（判断改善）、タイプ7（予測精度）
- **リスク**: Telegramレート制限、エージェント間の競合、実装複雑性
- **保留理由**: Reflexion 単独の効果測定（P3）が先。効果確認後に Multi-Agent 化を検討

- **実装難度**: High（monthly_leaderboard.py + Ghost 自動公開 + X 宣伝自動化）

### H5-EXEC: Ghost Members 有料化準備

- **問題**: 無料コンテンツのみで収益ゼロ
- **重要度**: Phase 2 の主要収益源

- **学習価値**: タイプ9（オペレーター意図整合）— 収益化はNaotoのミッション直結

- **前提条件**: Leaderboard（H4）稼働後
